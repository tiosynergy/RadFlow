Скоуп Stage 1. RadFlow не дзвонить і не пише пацієнтам автоматично. Продукт максимально полегшує адміну цю роботу: формує впорядкований Call List і дає сценарій дій при кожному збої. Автоматичні комунікації (Viber / Telegram / SMS / Email / Push) — опція Stage 2. Каталог послуг формується через завантаження прайс-листа клініки (Word / Excel / PDF / веб-посилання) → AI-обробка → підтвердження адміністратором.

1. Огляд платформи та RBAC

RadFlow — єдина SaaS-платформа. Усі ролі входять в один домен; кожна клініка — окремий tenant з ізоляцією даних (Row-Level Security у PostgreSQL). Адаптивний UI — десктоп або мобільний браузер, без окремих застосунків.

2. Адміністратор реєстратури
Налаштування · Черга · Call List · Інциденти

2.1 Первинна реєстрація та Setup Wizard (5 кроків)

As-Built: при першому вході показується модальне вікно первинної реєстрації (Клініка → Кабінети → Адміністратор → Готово). Повне налаштування — у «Майстрі налаштування», доступному будь-коли для правок.
Реєстрація→Прайс-лист (AI)→Апарати/кабінети→Графік→Персонал + Go Live
#ДіяЩо бачить / робитьРезультат
1РеєстраціяНазва клініки, email, пароль. Один екран.Акаунт створено, tenant активовано.
2Прайс-листЗавантажити Word/Excel/PDF або веб-посилання — AI-парсинг формує каталог. Fallback: ручна таблиця, якщо парсер не впорався.Каталог послуг готовий, рядки з низькою впевненістю позначені для перевірки.
3Апарати та кабінетиДодати апарат: назва, тип (МРТ/КТ), кабінет. Кнопка «+ Ще». Операційні параметри (тривалість, контраст, підготовка, пріоритет) пропонуються автоматично — адмін підтверджує.Кабінети збережені; Scheduler може планувати слоти.
4Графік роботиДні тижня + години. Шаблон Пн–Пт 8:00–18:00. Теплова карта тижня.Розклад збережено, слоти згенеровано.
5Персонал + Go LiveЗапросити радіолога (email-invite). «Запустити кабінет» + конфетті.Кабінет активний, перші записи можливі.

2.2 Щоденна робота (3 кроки)

#ДіяЩо бачить / робитьРезультат
1Відкрити Дошку чергиПеремикач: Усі кабінети (картки 7 кабінетів) або конкретний кабінет (черга лише його пацієнтів). Колір = статус. Календар та завантаженість праворуч-зверху, швидкий колл-лист під ними.Повний стан черги одним поглядом, real-time.
2Call List — обдзвінШвидкий колл-лист справа або повний екран. Розгортання пацієнта: кабінет, тип, вік, дія. Статуси: ✓ Підтверджено / ✗ Не відповів / ↩ Передзвонити / Відмова. Підтверджений зникає зі швидкого списку.Завтрашня черга підтверджена; статуси синхронізовано.
3Новий запис / перезапис«Новий запис» → Smart Scheduler пропонує найближчий слот; зайняті — заблоковані. Інцидент → «Перенести день».Запис збережено, пацієнт у черзі.

2.3 Завершення процедури (ADMIN-PROC-01)

Радіолог або адміністратор підтверджує результат після кожної процедури. Статус оновлюється для всіх ролей одночасно (Supabase Realtime).

#ДіяЩо бачить / робить
1Відкрити пацієнтаКлік по запису в черзі або кнопка «Завершити процедуру» на картці кабінету.
2Обрати результат● Успішно (→ Виконано) / ○ Не відбулось → dropdown причини (стан пацієнта / технічна несправність → лінк на Інциденти / інше).
3ПідтвердитиRealtime-оновлення статусу для Admin · Radiologist · CEO · лікаря-направника.
As-Built — виправлення статусу. У розгорнутому рядку черги є сегмент-контрол «Змінити статус» (Очікує / В кабінеті / Виконано / Не відбулось). Будь-який статус можна виправити у разі помилкового натискання — переведення «В кабінеті» блокується, якщо кабінет зайнятий іншим пацієнтом.

3. Радіолог / Завідувач відділення
Моя черга · Картка пацієнта · Статус
As-Built кабінет радіолога. Радіолог бачить чергу лише тих кабінетів, у яких його авторизував Адміністратор. Картка пацієнта відкривається у модальному вікні. Статус дослідження змінюється сегмент-контролом. Протокол/архів на цьому етапі прибрані з ролі.
#ДіяЩо бачить / робитьРезультат
1Відкрити чергу«Моя черга»: пацієнти авторизованих кабінетів, сортування — в кабінеті → CITO/терміново → час. Фільтри статусів.Видно навантаження на сьогодні.
2Фільтр за кабінетомКлік по кабінету в сайдбарі → лише його пацієнти. «Моя черга» — усі авторизовані.Сфокусована черга кабінету.
3Картка пацієнтаКлік по рядку → модальне вікно: процедура, кабінет/апарат, час, контраст, вага, лікар-направник із телефоном.Повний клінічний контекст.
4Зміна статусуСегмент: Очікує / В кабінеті / Виконано / Не відбулось. Живий таймер у кабінеті. Зміна синхронізується з дошкою адміна.Статус оновлено для всіх ролей.
5ПриміткиВнутрішня нотатка радіолога з автозбереженням.Контекст збережено для команди.

Завідувач відділення — додатково: аналітика по апаратах, перегляд протоколів усіх радіологів (read-only), експорт статистики (CSV).
