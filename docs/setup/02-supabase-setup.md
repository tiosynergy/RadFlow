# Шаг 2 — Supabase: база данных + вход

Это «фундамент» приложения. После него у RadFlow появятся: регистрация центра, вход сотрудников и изоляция данных между клиниками (multi-tenant). Код уже написан — вам нужно создать проект Supabase, выполнить SQL и вставить два ключа.

Время: ~15 минут. Программировать ничего не нужно.

---

## 1. Создать проект Supabase

1. Откройте <https://supabase.com> → **Start your project** → войдите через GitHub.
2. **New Project**. Заполните:
   - **Name:** `radflow`
   - **Database Password:** придумайте надёжный и **сохраните** (понадобится при бэкапах).
   - **Region:** ближайший к Украине — `Central EU (Frankfurt)`.
3. Нажмите **Create new project** и подождите ~2 минуты, пока он поднимется.

## 2. Скопировать ключи

В проекте: **Project Settings** (шестерёнка) → **API**. Скопируйте два значения:

- **Project URL** — вида `https://xxxxxxxx.supabase.co`
- **anon public** ключ (длинная строка под *Project API keys*)

> `service_role` ключ — секретный, понадобится позже. Пока можно не трогать.

## 3. Вставить ключи в проект

Откройте файл `.env.local` в корне `RadFlowDev` и впишите значения:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi... (ваш anon key)
```

Сохраните файл. Этот файл не попадает в Git (он в `.gitignore`).

## 4. Создать таблицы (выполнить SQL)

1. В Supabase слева откройте **SQL Editor** → **New query**.
2. Откройте файл `supabase/migrations/0001_init.sql` из проекта, скопируйте **всё** его содержимое, вставьте в SQL Editor и нажмите **Run** (или Ctrl/Cmd+Enter).
3. Создайте ещё одну **New query**, выполните так же `supabase/migrations/0002_setup.sql` (поля профиля клиники, расписание оборудования, политика на изменение клиники — нужно для Мастера настройки).
4. После каждого запуска должно появиться `Success. No rows returned`.

> Запускайте миграции по порядку: сначала `0001_init.sql`, затем `0002_setup.sql`. Они безопасны для повторного запуска.

Что создалось: таблицы `clinics`, `profiles`, `services`, `rooms`, `queue_entries` (+ связь исследований), политики безопасности RLS (изоляция по клинике) и триггер, который при регистрации автоматически заводит клинику и профиль администратора. Миграция `0002` добавляет поля профиля клиники и расписание кабинетов для Мастера настройки.

## 5. Настроить вход по email

1. **Authentication** → **Sign In / Providers** → убедитесь, что **Email** включён.
2. **Authentication** → **URL Configuration**:
   - **Site URL:** `http://localhost:3000`
   - **Redirect URLs:** добавьте `http://localhost:3000/auth/callback`
   - (Позже, после деплоя на Vercel, добавьте сюда же боевой адрес `https://<ваш-проект>.vercel.app/auth/callback`.)

> На время разработки можно отключить подтверждение email, чтобы входить сразу:
> **Authentication → Sign In / Providers → Email → Confirm email = OFF**. Для боевого режима верните ON.

## 6. Установить пакеты и запустить

В терминале VS Code (в папке `RadFlowDev`):

```bash
npm install
npm run dev
```

`npm install` подтянет добавленные пакеты `@supabase/supabase-js` и `@supabase/ssr`.

## 7. Проверка

1. Откройте <http://localhost:3000/register> → зарегистрируйтесь (логин, email, телефон, пароль).
2. Если подтверждение email включено — откройте письмо и нажмите ссылку (вернёт в приложение). Если выключено — сразу зайдите на <http://localhost:3000/login> и войдите.
3. После входа админ попадает на `/queue` (Дошка черги); если клиника ещё не настроена — на `/setup`. Радиолог → `/radiologist`, лікар-направник → `/referral`.
4. В Supabase → **Table Editor** → таблицы `clinics` и `profiles` появилась новая запись.
5. Попробуйте открыть `/queue` в режиме инкогнито (без входа) — должно перебросить на `/login`. Значит защита маршрутов работает.

Готово — фундамент собран. Следующий шаг по плану: **Майстер налаштування (Setup Wizard)**, затем **Дошка черги**.

---

### Если что-то не работает

- **Кнопка регистрации пишет «Supabase ще не налаштований»** — не заполнен `.env.local` или не перезапущен `npm run dev` после правки `.env.local`.
- **«Невірний email або пароль» сразу после регистрации** — включено подтверждение email, сначала подтвердите письмо (или отключите Confirm email на время разработки).
- **Ошибка о redirect при переходе по письму** — проверьте, что `http://localhost:3000/auth/callback` добавлен в Redirect URLs (шаг 5).
