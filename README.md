# RadFlow

**Интеллектуальное управление очередью для центров лучевой диагностики (МРТ/КТ).**
SaaS-платформа: запись пациентов, доска очереди в реальном времени, интерфейс радиолога,
обзвон, AI-перепланирование при инцидентах, дашборд руководителя. Multi-tenant (несколько клиник).

## Технический стек

| Слой | Технология |
|------|-----------|
| Фронтенд + бэкенд | Next.js 15 (App Router) + TypeScript + Tailwind + shadcn/ui |
| База данных, авторизация, файлы | Supabase (PostgreSQL + RLS) |
| Реальное время | Supabase Realtime |
| Автоматизации / AI-логика | n8n Cloud (сценарии + AI-узлы) |
| AI-мозг | OpenAI / Anthropic API |
| Хостинг | Vercel (авто-деплой из этого репозитория) |
| Email | Resend |
| Мониторинг | Sentry |
| Инструмент сборки | Claude Code в VS Code |

## Быстрый старт

> Полная пошаговая инструкция — в `docs/plan/RadFlow_План_реализации_MVP_v5_радиолог.docx`.

1. Установите Node.js (LTS), VS Code и расширение **Claude Code**.
2. Склонируйте репозиторий и откройте папку в VS Code.
3. Скопируйте `.env.example` → `.env.local` и впишите ключи (Supabase, AI, n8n, Resend).
4. Попросите Claude Code создать проект Next.js в этой папке (Шаг 1 плана) и далее идите по шагам.
5. Подключите репозиторий к Vercel — деплой произойдёт автоматически при `git push`.

## Безопасность

- **Ключи только в `.env.local`** (и в Environment Variables на Vercel). Никогда не коммитьте секреты.
- **RLS включён на каждой таблице** с `tenant_id` — данные одной клиники не видны другой.

Интерфейс радиолога («Моя очередь») · Запись пациента + Портал направляющих ·
Call List · Инциденты + AI-перепланирование (n8n) · CEO Dashboard.
