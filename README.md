# RadFlow

**Интеллектуальное управление очередью для центров лучевой диагностики (МРТ/КТ).**
SaaS-платформа: запись пациентов, доска очереди в реальном времени, интерфейс радиолога,
обзвон, AI-перепланирование при инцидентах, дашборд руководителя. Multi-tenant (несколько клиник).

## Технический стек

| Слой | Технология |
|------|-----------|
| Фронтенд + бэкенд | Next.js 15 (App Router) + TypeScript + Tailwind |
| База данных, авторизация, файлы | Supabase (PostgreSQL + RLS) |
| Реальное время | Supabase Realtime |
| Автоматизации / AI-логика | n8n Cloud (сценарии + AI-узлы) |
| AI-мозг | OpenAI / Anthropic API |
| Хостинг | Vercel (авто-деплой из этого репозитория) |
| Email | Resend |
| Мониторинг | Sentry |

## Текущее состояние (Шаг 1)

Каркас Next.js создан, приложение запускается. Маршруты:

- **`/` — Дошка черги адміністратора** (главная). Актуальный прототип из
  `D:\Проект\HTML\RadFlow`: расписание по кабинетам, статусы в один клик, фильтры/поиск,
  мини-календарь, загруженность, колл-лист, модалки, CITO-баннер, симуляция потока.
- Связанные экраны (доступны по навигации из доски): **Колл-лист**, **Інциденти**,
  **Кабінет радіолога**, **Майстер налаштування**.
- **`/register`** — регистрация (соц-вход Google/Facebook/X + email-форма с валидацией).

> Данные пока **мок** (статика в `public/board`). На Шаге 2 плана они заменяются на Supabase.

## Локальный запуск

```bash
npm install      # установить зависимости (один раз)
npm run dev      # http://localhost:3000
```

http://localhost:3000 открывает Дошку черги; переходы по сайдбару ведут на остальные экраны.

## Как обновить экраны из прототипа

Все экраны — это файлы прототипа из `D:\Проект\HTML\RadFlow`, разложенные в `public/board/`.
Страницы (`radflow-*.html`) связаны относительными ссылками + `<base href="/board/">`,
а `next.config.mjs` отдаёт `/` (и `/queue`) как `radflow-queue-board.html`.

Чтобы подтянуть свежую версию прототипа — одна команда:

```bash
npm run sync-board
# или с другим путём к прототипу:
npm run sync-board -- "D:\путь\к\RadFlow"
```

Скрипт (`scripts/sync-board.mjs`) копирует данные и стили как есть, транспилирует все
`*.jsx` → `public/board/*.js` через esbuild (без браузерного Babel) и патчит HTML-страницы
(добавляет `<base>`, переводит ссылки `.jsx`→`.js`, убирает Babel, ставит production-сборку React).

## Деплой на Vercel

1. Скопируйте `.env.example` → `.env.local` и впишите ключи (Supabase, AI, n8n, Resend) — для локальной работы.
2. На [vercel.com](https://vercel.com) → **Add New… → Project** → выберите репозиторий `tiosynergy/RadFlow` → Import.
   Vercel сам определит Next.js.
3. В **Settings → Environment Variables** добавьте те же ключи из `.env.example` (для Production/Preview/Development).
4. **Deploy**. Далее каждый `git push` в `main` пересобирает прод автоматически.

## Безопасность

- **Ключи только в `.env.local`** (и в Environment Variables на Vercel). Никогда не коммитьте секреты.
- **RLS включён на каждой таблице** с `tenant_id` — данные одной клиники не видны другой.

## Структура

```

├─ app/                  # Next.js App Router
│  ├─ register/page.tsx  # /register — регистрация (рендерит RegisterPage)

│  ├─ radflow-call-list.html     # Колл-лист
│  ├─ radflow-incidents.html     # Інциденти
│  ├─ radflow-radiologist.html   # Кабінет радіолога
│  ├─ radflow-setup-wizard.html  # Майстер налаштування
│  ├─ *-data.js                  # данные (мок)
│  └─ *.js, *.css                # компоненты (транспилировано из .jsx) + стили
├─ scripts/sync-board.mjs # синхронизация прототипа → public/board
├─ next.config.mjs · tsconfig.json · tailwind.config.ts · package.json
├─ .env.example          # шаблон переменных окружения
├─ .gitignore
├─ README.md
└─ docs/                 # справочные материалы (на деплой не влияют)
```
